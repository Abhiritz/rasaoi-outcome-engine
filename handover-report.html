DIET-001

Dish Dietary Taxonomy — Full Stack

Resolved

Client summary

Every dish now has a clear primary diet class (vegan, vegetarian, eggetarian, non-veg) plus optional religious modifiers (Jain, Halal, Jhatka, Kosher). These fields flow from menu ingest through the database into Reading-page recommendations — tags-first gating with regex fallback — so a vegetarian intent never surfaces Tandoori Chicken, and eggetarian correctly allows eggs but blocks meat.

Date resolved

18 June 2026

Coverage

144 dishes re-tagged · 8 venues

Impact area

Ingest, gates, Reading UI, Lab

Taxonomy boundaries

  • Primary class — exactly one: vegan ⊂ vegetarian; eggetarian allows eggs; non_veg for meat/fish
  • Modifiers — jain, halal, jhatka, kosher (halal and jhatka never on same dish)
  • menu_items — now carries diet fields end-to-end (no tag loss at commit)

DIE-001

Hard-Exclusion Gate for Strict Dietary Restrictions

Resolved

Client summary

When a guest has a strict religious or lifestyle diet — for example "my friend is a Jain, it is his birthday" — Rasaoi enforces a zero-tolerance safety gate at every layer. Non-compliant dishes are removed before scoring and from every nested recommendation slot (hero + alternates + triple-outcome arrays). Dal Tadka, Tandoori Chicken, and other violators cannot slip through cuisine-bank fallbacks. Birthday context never overrides Jain rules.

Date resolved

27 May 2026

Status

RESOLVED

Impact area

Intent parsing & recommendation safety

Gatekeeper architecture

  • Intent layer — parse-intent extracts filters.dietary (jain, vegan, halal, kosher); celebratory keywords only raise dials.context, never erase dietary rules.
  • Transcript guard — server-side regex cross-check ensures Jain/vegan/halal/kosher mentions survive even if the LLM omits them.
  • Veda gatekeeper — passesStrictDietaryGate() runs on each venue's dish blob; violators are dropped before any score math.
  • Nested array gate — buildTripleOutcome() filters every dish slot; Jain bank replaces generic Indian fallbacks; ambiguous dishes (Dal Tadka) blocked unless explicitly Jain.
  • Mock fixtures — Full menu_items arrays with explicit ingredient keywords; Jain kitchens carry Jain Paneer Tikka, Jain Dal Makhani, Fresh Fruit Salad.

Jain conflict — before vs after

Before

Query: "my friend is a jain, it is his birthday" → Tandoori Chicken ranked first (high context + Indian purity boosts).

After

Same query → Tandoori and Dal Tadka excluded at every layer; all three outcome slots show Jain-compliant dishes only (Jain Paneer Tikka, Jain Dal Makhani, Fresh Fruit Salad).

npm test -- --run → 27 passed (dietary + veda + pairings)
