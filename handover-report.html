ARCH-001

Pure Agentic Generation Loop

In Progress

Client summary

Rasaoi has pivoted from erratic database and mock-file lookups to a Pure Agentic Generation Loop. Veda (Gemini) now synthesizes the entire recommendation — restaurants, menus, prices, match scores, and context-aware descriptions — in one call. Jain queries receive explicitly compliant synthetic venues; Thai queries receive purely Thai instances; birthday and wellness context is woven into every description. No Dal Tadka or Tandoori Chicken can leak from stale fixture fallbacks.

Date started

11 Jun 2026

Git branch

feature/agentic-generation-loop

Impact area

Core recommendation architecture

New pipeline

  • Ask Veda → user transcript sent to parse-intent edge function.
  • Gemini generates → 5 synthetic restaurants with full menu_items, match_score, and constraint-aware why text.
  • Veda pass-through → veda.ts maps agent JSON to UI types — no token scoring, no mock lookup.
  • Reading page → renders agent payload from session; Supabase restaurant fetch and Google Places search bypassed.

Before vs after

Before

Intent parser → DB/mock lookup → token scoring → cuisine-bank fallbacks. Mixed Jain + birthday queries broke across layers.

After

Single Gemini call generates fully compliant synthetic restaurants. Semantic constraints enforced at generation time.
