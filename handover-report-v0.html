CRS-001

Explicit Cuisine Intent Mis-Routed to Indian Options

Resolved

Client summary

When diners asked for a specific cuisine (for example Thai), Rasaoi could still highlight Indian restaurants and dishes like Tandoori Chicken. The app now interprets cuisine requests more accurately and prioritizes restaurants that genuinely match what you asked for — so recommendations feel intentional, not random.

Date resolved

27 May 2026

Status

RESOLVED

Impact area

Recommendations & intent

1. Technical Root Cause Analysis

User symptom

Guests requesting Thai (sometimes alongside phrases like “for my partner” or “nearby”) received hero suggestions for Indian venues — often featuring Tandoori Chicken — despite no Indian cue in the request.

System conflict

Layer Expected Observed failure
AI intent parser Extract cuisine: Thai Occasional default / hallucinated Indian filters
Veda ranker Cuisine intent drives hero pick Cuisine filter never passed into scoring
UI sort Thai venues surface first Soft re-sort lost to high purity/promo scores

Root cause

The AI parser lacked server-side validation, so incorrect filters could pass through unchanged. Separately, the ranking engine rewarded purity and wellness attributes (where Indian curated data is strong) without penalizing cuisine mismatches — allowing Indian signature dishes to mathematically outrank true Thai matches.

2. Resolution & Verification

Fix applied

  • Stricter AI instructions — explicit rules: never invent cuisines/dishes; relative phrases must not override food keywords.
  • Server validation — transcript cross-check corrects or fills cuisine; strips conflicting dish hallucinations.
  • Ranking gates — matching cuisine +48 points; mismatch −42 points before other wellness boosts apply.
  • Full pipeline wiring — cuisine filter now flows from parser → ranker on every recommendation path.
